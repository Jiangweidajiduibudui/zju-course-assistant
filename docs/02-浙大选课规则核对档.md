# 02 · 浙大选课规则核对档

- **版本**:v2.0(2026-07-08)
- **权威范围**:选课阶段与轮次、志愿组/志愿顺序/录取规则、本专业与辅修等优先级、时间冲突/考试冲突/学分与门数限制;每条结论的来源、核验时间、核验方式、置信度;开放问题。
- **地位**:**志愿、顺位、录取风险及相关推荐设计,必须等本文件闭环后再重新设计**(D01)。本档未闭环期间不发布产品(D06)。

---

## 1. 冻结声明(D01)

在本档 §4 开放问题全部闭环前,以下设计**冻结**,不得实现、不得以占位方式上线:

1. 志愿数据模型(含任何"双层志愿"临时模型);
2. 顺位排序策略(含"同槽默认按最容易选中排序"等假设);
3. 录取风险的算法、档位与展示(产品内一律显示"**暂不可评估**",不用"稳/悬/危"等无依据档位);
4. 一切依赖上述规则的推荐器行为。

解冻流程:开放问题闭环 → 在 [06-决策记录](06-决策记录与版本变更.md) 追加新决策条目 → 更新本档结论表 → 相关设计再入 [01-PRD](01-产品需求文档PRD.md) 与 [04-架构](04-AI与技术架构设计.md)。

## 2. 核验方法要求

每条结论必须记录:**来源 / 核验时间 / 核验方式 / 置信度(高·中·低)**。可接受的核验方式(可信度从高到低):

1. 教务处官方文件 / 选课通知原文;
2. 教务处或学院官方答复(留存截图/邮件);
3. 选课窗口期内登录态实测(抓包/页面行为);
4. 多名近期亲历者交叉印证;
5. 公开项目代码中的机制推断(**仅作交叉参考,不得单独作为结论依据**)。

## 3. 已确认结论

> "口径确认"= 产品负责人基于亲历经验确认,尚无书面来源,置信度按中处理,beta 前应升级核验方式。

| # | 结论 | 来源 | 核验时间 | 方式 | 置信度 |
|---|---|---|---|---|---|
| R-01 | 选课自由:培养方案必修四年内修完即可毕业,不必按推荐课表 | 产品负责人 | 2026-07 | 口径确认 | 中 |
| R-02 | 阶段结构:预置(选课前完成)→ 随机选课阶段 → 补选(申请制) | 产品负责人 | 2026-07 | 口径确认 | 中 |
| R-03 | 预置 = 已选中、直接占用容量;退预置后本轮容量不释放;无"接受"动作,只能退 | 产品负责人 | 2026-07 | 口径确认 | 中 |
| R-04 | 录取机制为顺序志愿:人数<容量全录;>容量时优先录取有优先级者(本专业/辅修),再随机 | 产品负责人 | 2026-07 | 口径确认(细则见 O-01/O-02) | 中 |
| R-05 | 系统对每班暴露:余量、容量、本专业待选、总待选(实时);上课时间为位域编码 | 登录态实测 | 2026-07 | 页面/接口观察(字段语义核验状态见 [03](03-数据接口契约.md)) | 中 |
| R-06 | 选课阶段内志愿可随时修改(→"定稿-录入-事后检查"流程成立,录入错误可补救) | 产品负责人 | 2026-07 | 口径确认 | 中 |
| R-07 | zdbk 选课页面当前暴露 3 个志愿位 | 登录态实测 | 2026-07-08 | 页面观察 | 高 |
| R-08 | 志愿分组可能由服务器字段动态决定,而非固定双层结构 | 登录态实测 | 2026-07-08 | 页面/接口观察(仅为迹象,结构未定论 → O-03) | 低 |

## 4. 开放问题(闭环前冻结相关设计)

| # | 问题 | 阻塞什么 | 建议核验方式 |
|---|---|---|---|
| O-01 | 顺位档内录取细则:某志愿档申请>余量时,是否"本专业优先全录,剩余在非本专业中随机" | 录取风险算法 | 官方文件/教务答复 + 窗口期实测 |
| O-02 | 随机触发条件:本专业待选≤余量是否必中;随机是否仅发生在"非本专业+名额不足" | 录取风险算法 | 同上 |
| O-03 | 真实志愿分组与结构:是否同时存在"同课不同班志愿"与"同一时段不同课顺位";分组是否由服务器字段动态决定 | 志愿数据模型、待选池与方案的数据结构 | 窗口期抓包(服务器字段)+ 官方说明 |
| O-04 | 同槽顺位默认排序策略(依赖 O-01/O-02/O-03) | 推荐器顺位行为 | 规则闭环后由产品决策 |
| O-05 | 本专业判定口径(主修/辅修/转专业) | 录取风险算法 | 教务答复 |
| O-06 | 同轮能录取几门;轮次结构细节 | 方案生成约束 | 官方通知 + 实测 |
| O-07 | 时间冲突是否被系统硬性禁止提交 | 求解器约束建模 | 窗口期实测 |
| O-08 | 选课学分/门数上限 | 求解器约束建模 | 官方文件 |
| O-09 | 录取风险的算法、档位阈值与校准方法(规则闭环后仍需产品负责人专项决策,见 06 O-Q3) | 风险展示 | 产品决策 + 模拟校准 |
