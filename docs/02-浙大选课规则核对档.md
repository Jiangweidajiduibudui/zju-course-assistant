# 02 · 浙大选课规则核对档

- **版本**:v2.4(2026-07-09)
- **权威范围**:预置课程、第一/二轮概率筛选、课程面向对象优先级、志愿分组与顺序、容量释放口径，以及第一版推荐模型的简化假设。
- **适用边界**:产品只建模第一、二轮概率筛选；第三/四轮、补选、低分定点选课、退课与放弃修读不在第一版范围内(D30)。
- **地位**:志愿数据模型和顺位规则已闭环；未经校准的录取概率、风险档位和阈值仍冻结，产品继续显示“暂不可评估”(D01、D30)；时间、考试与学分硬约束按 D37/D38 执行。

---

## 1. 建模范围与冻结边界

### 1.1 第一版纳入

1. 选课前已完成的预置课程;
2. 第一、二轮概率筛选;
3. 课程面向对象优先;
4. 同一课程和同一时间段两类志愿约束;
5. 第一、二、三志愿顺序和同档随机筛选;
6. 当前轮退掉预置课后容量不释放、下一轮释放的口径;
7. 第一版用户方案的考试硬冲突、必填学分上限和缺失硬字段处理。

### 1.2 第一版不纳入

- 第三、四轮的规则与操作;
- 补选审核、低分定点选课、退课、放弃修读;
- 已获学分、当轮已选学分或其他学分因素对录取优先级的影响;
- 课程面向对象之内的额外细分优先级;
- 未经校准的具体中签概率、“稳/悬/危”档位与阈值。

## 2. 核验与口径规则

每条系统事实应记录:**来源 / 核验时间 / 核验方式 / 置信度(高·中·低)**。证据优先级:

1. 本科生院官方管理办法、当期选课通知;
2. 教务处或学院官方答复;
3. 选课窗口期登录态实测(抓包/页面行为);
4. 产品负责人基于亲历经验确认的产品口径;
5. 非官方选课指南或公开项目的交叉参考。

> “产品负责人确认”可作为第一版建模口径，但不伪装成官方书面规则。

| # | 结论 | 来源 | 核验时间 | 方式 | 置信度 |
|---|---|---|---|---|---|

| R-02 | 预置课程在选课开始前已选中并占用容量;没有“接受预置”动作，学生只能退改 | 官方当期通知 + 产品负责人 | 2026-07 | 文档核对 + 口径确认 | 高 |
| R-03 | 退掉预置课程后，容量在**当前轮不释放**，在**下一轮释放** | 产品负责人 | 2026-07-08 | 亲历口径确认 | 中 |
| R-04 | 第一、二轮是轮次结束后统一处理的概率筛选，不依据提交时间先后 | 官方管理办法 + 非官方指南交叉参考 | 2026-07 | 文档核对 | 高 |
| R-05 | 报名人数小于容量时全部录取;超过容量时按“课程面向对象优先 → 志愿顺序优先 → 同一志愿档内随机”处理 | 官方管理办法 + 产品负责人 | 2026-07-08 | 文档核对 + 口径确认 | 高 |
| R-06 | 优先级判定依据是**课程面向对象**;不是所有课程都区分本专业，通识选修、体育等所有学生优先级相同的课程直接按志愿顺序筛选 | 官方管理办法 + 产品负责人 | 2026-07-08 | 文档核对 + 口径确认 | 高 |
| R-07 | 同一课程最多填 3 个志愿;同一时间段最多填 3 个志愿;两类约束同时生效 | 产品负责人 + 登录态页面观察 | 2026-07-08 | 口径确认 + 页面观察 | 高 |

| R-09 | 页面“待选人数”包含第一、二、三志愿的申请，不能直接当作第一志愿人数 | 产品负责人 | 2026-07-08 | 登录态观察口径确认 | 高 |
| R-10 | 选课轮次内志愿可修改;已筛选录取的课程如改选，需重新参与筛选 | 官方当期通知 + 产品负责人 | 2026-07 | 文档核对 + 口径确认 | 高 |

## 4. 第一版模型决策(D30)

| # | 决策 | 目的/代价 |
|---|---|---|
| M-01 | 只建模第一、二轮概率筛选 | 避免将第三/四轮、补选和低分定点等不同机制混入同一模型 |
| M-02 | 课程面向对象内不再细分优先级 | 第一版不模拟年级、重修等额外次级顺序 |
| M-03 | 不考虑已获学分或当轮已选学分对筛选的影响 | 学分不进入优先级或录取概率计算 |
| M-04 | 待选人数虽包含三个志愿档，第一版对各顺位使用同一聚合待选口径 | 这是显式简化，只能用于相对负载比较，不得宣称为精确概率 |
| M-05 | 志愿数据同时校验“同课程最多 3 个”和“同时间段最多 3 个”，每个顺位必须是具体教学班 | 两类志愿约束叠加，不采用临时“双层志愿”猜测模型 |
| M-06 | 同一课程 2 个及以上教学班形成课程志愿组；课程组优先于时间槽志愿组 | 发生冲突时优先保留课程志愿组，时间槽组失效 |
| M-07 | 上课时间重叠不直接判无解；考试时间重叠按不同课程硬无解处理 | 上课重叠交给志愿排序与 LLM 软判断；考试冲突保证最终方案可执行 |
| M-08 | session 必填学分上限；第一版不设置课程门数上限 | 每人学分上限受多因素影响，由用户自己填写；门数上限触发概率低，后续再加入 |

## 5. 筛选规则摘要

```text
第一/二轮截止
  → 按课程面向对象划分优先级
  → 高优先级学生先处理
  → 同一优先级内按第一、第二、第三志愿顺序处理
  → 某档申请人数 ≤ 剩余容量:全部录取
  → 某档申请人数 > 剩余容量:同档随机分配
  → 已被同组高顺位志愿录取的学生不再处理该组低顺位
```

这是产品的确定性规则模型，不等于已获得教务系统的完整内部实现。
