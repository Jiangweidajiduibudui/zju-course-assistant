## 已被 08 修正的关键点

- 正式 Demo 一次直接跑通，不设置对外 mock 版主干。
- 选课数学模型集中在 `selection-model` 模块。
- 上课时间重叠不直接判无解；考试时间重叠按 D37 硬校验。
- 学分上限由用户必填；缺失学分的教学班留在待选池。
- LLM 采用组内排序 + Top10 完整方案比较的两阶段流程。
- 所有 LLM 请求走同源 Fastify 代理并做 SSRF 防护。
- UI 模仿 zdbk 的待筛选志愿和预期课表心智模型。
