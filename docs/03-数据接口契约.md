# 03 · 数据接口契约

- **版本**:v2.3(2026-07-08)
- **权威范围**:zdbk 与 chalaoshi 的端点/方法/参数、实测字段与语义、示例响应要求、缓存与刷新策略、异常处理、只读白名单与写接口黑名单、最近核验日期。
- **总原则**:本文所有内容以**真机抓包/实测为准**;公开项目(Lazuli/Quantum)推断只能作为交叉参考,不能替代实测与固定样例。字段未经核验前,**不能凭名称直接解释语义**。

---

## 1. 读写边界(advise-only 铁律)

- **Demo 只读边界**:网站后端仅主动抓取本文明确列出的 chalaoshi 公开资源;zdbk 数据由用户手动输入/导入或 Demo fixture 提供(D31、D32)。
- **写接口黑名单**:**禁止调用选课、退选、调序等一切业务写接口**。实现层面采用白名单策略——任何不在白名单内的端点(尤其 POST 修改类)一律禁止;新增读接口须先补入本契约。
- 独立网站不注入 zdbk 页面，不索取/保存统一身份认证密码、Cookie 或会话令牌。

- **模块参数**:`gnmkdm=N253530`(2026-07 登录态验证有效)、`su=<学号>`。

### 2.3 课程分类

**分类必须动态读取,不得硬编码**:分类集合(培养方案分类、通识各类等)随学期/政策变化,2026-07 实测已与旧文档枚举不同。实现:运行时从选课页面/接口读取分类清单;"选课搜索引擎"入口(页面内 `id=searchTool`,`data-dl=xk_9`,2026-07 实测存在)作为自由搜索通道。

### 2.4 异常处理契约

| 情形 | 识别 | 行为 |
|---|---|---|
| 登录失效 | 响应重定向至 CAS/登录页或特征状态码 | 通知 UI"请先登录";不重试轰炸 |
| 非选课期 | 接口返回空/特征提示 | 明示;信息类功能不受影响 |
| 空数据 | 正常响应但列表为空 | 如实展示"无数据",不当作错误 |
| 接口变更 | 字段缺失/解析失败 | 明示"数据获取失败(接口可能已变更)";上报待人工核查;**不静默吞错** |

### 2.5 缓存与刷新

- **Demo**:网站不自动刷新 zdbk 余量、待选人数或正式状态;显示用户导入时间，生成推荐/最终确认前要求用户确认时效性(D20、D31)。
- **长期**:如 O-Q11 确定合法只读通道，再恢复进入 session、基线同步、生成推荐与一致性检查时的定向刷新。

## 3. chalaoshi 抓取与数据接口(D32)

- **最近核验**:2026-07-08(实测,下表逐项标注)。
- **域名与可用性风险**:chalaoshi 官方 README 声明 **`chalaoshi.de` 已被墙,建议科学上网**;另提供 arpa 备用域名(可用性无保证)。→ 产品必须把"chalaoshi 不可达"当作**常态场景**设计(缓存优先、明确降级,见 [01-PRD](01-产品需求文档PRD.md) FR-12);域名应做成可配置项。
- **Demo 访问策略**:由网站后端按用户请求直接抓取;按教师 ID 去重并发、限频、超时、失败显式化并优先命中后端缓存，避免高频触发 Cloudflare。

### 3.2 Demo 缓存策略(D28、D32)

- **教师索引(search.json)**:网站后端首次需要时抓取，并使用共享缓存;不随前端构建产物打包。
- **教师详情与评论**:用户点开时由后端按需抓取，共享缓存 **3 天**。
- **展示要求**:显示数据更新时间;支持强制刷新;使用过期缓存时**显著标记**。

### 3.3 长期镜像与向量索引(D32)

- 长期考虑定期抓取并建立镜像数据层:原始响应/页面快照 → 规范化教师、课程、评论记录 → 对外查询 API。
- 每条镜像记录保留 `sourceUrl`、`fetchedAt`、源内容版本/哈希和删除状态。
- 评论文本可生成向量索引，用于语义检索、去重和摘要取样;向量条目必须可回溯到规范化记录，不得成为独立事实源。
- 镜像更新频率、公开形态、embedding 模型和删除同步 SLA 待 06 · O-Q9 决策；向量数据库起步方案已确定为 PostgreSQL + pgvector(D33)，Demo 不启用。

### 3.4 数据合规立场(D03,权威决策见 06)

事实基础(来自 chalaoshi 官方仓库 README,2026-07-08 读取):**评论作者保留其内容的全部版权**,站方仅获得永久、不可撤回、非独占的许可。因此:

- 本项目**不宣称拥有数据**;不得使用"chalaoshi 没有数据版权"或"其声明必然无效"之类表述;
- 明确标注数据来源,始终保留原页链接;
- 提供**投诉、删除和停止收录机制**(渠道:本项目自身反馈入口;chalaoshi 侧可引导其官方 GitHub Issues / 邮箱);
- Demo 按需抓取并缓存，长期考虑镜像与向量索引;**不把取得站方授权作为上线前置条件——此为已接受的已知合规风险**(复审条件见 06 · D03/D32)。

## 4. 隐私与脱敏

- 所有留存的示例响应/固定样例必须**删除个人信息**(学号、姓名、班级名单等)。
- Demo 不采集 zdbk 密码、Cookie 或会话令牌;用户导入的样例在上传或持久化前必须脱敏(见 04 §隐私)。
