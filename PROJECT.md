# PROJECT.md · 项目公约

> 所有人开始开发前必须阅读。违反红线的 PR 不会被合并。

---

## 红线（不可违反）

1. **不调用 zdbk 写接口**：不能选课、退选、调序，也不能处理 zdbk Cookie / 密码 / token。白名单读接口见 `docs/03-数据接口契约.md`。
2. **不提交真实个人数据**：真实评论、学号、姓名、Cookie、API key 一律不进仓库。Mock 数据必须明确标注 `MOCK`。
3. **不展示录取概率**：统一显示"暂不可评估"，不做"稳/悬/危"。冻结原因见 `docs/06-决策记录与版本变更.md` D01。
4. **硬规则只写在 `selection-model`**：前端和后端不允许各写一份"简化版冲突判断"。客户端校验仅用于即时反馈，最终判定以后端 `selection-model` 为准。
5. **LLM 输出必须校验**：AI 返回的方案必须过 JSON Schema → 确定性终校验（Schema、池内性、锁定保持、冲突、覆盖）。直接信任 LLM 输出是禁止的（见 D25）。
6. **不随意升级依赖或加新包**：业务 PR 不顺手改依赖版本。新增包需要明确理由并在 PR 描述中说明。

## 技术决策速览

| 决策 | 编号 | 摘要 |
|---|---|---|
| 推荐核心 | D05 | 确定性求解工作流，不是 agent loop |
| LLM 角色 | D05 | 只做偏好结构化、评论理解、软排序、解释 |
| Chrome 写接口 | D09 | 允许 content script 注入增强 UI，但写接口禁令不变 |
| Session 模型 | D08 | 用户手动创建，基线快照，不复用历史 |
| 数据新鲜度 | D20 | 生成/重新优化/确认前必须强制刷新关键数据 |
| 方案数量 | D16 | 只输出**一份**完整推荐方案 |
| 规则优先级 | D17 | 手动锁定 > 局部硬约束 > 全局硬约束 > 局部软偏好 > 全局软偏好 |
| LLM 兼容 | D10 | OpenAI 兼容端点均可；首次配置做能力检测 |
| 无 key 降级 | D21 | 无 key 可用全部功能，仅 AI 排序和评价摘要不可用 |
| 隐私同意 | D23 | 同意本地存储/读取；同意 LLM 数据发送；不同意则无法进入 |

完整决策记录见 `docs/06-决策记录与版本变更.md`。

## 开发环境

```bash
# 安装依赖
pnpm install

# 启动开发服务器
pnpm dev

# 类型检查
pnpm lint

# 构建
pnpm build

# 运行 E2E 测试
pnpm test

# 完整验证（lint + build + test）
pnpm verify
```

---

## 目录约定

```text
src/shared/contracts/   → 公共类型、错误码（前后端共享、单一数据源）
src/domain/selection-model/ → 选课数学模型（硬约束、冲突、学分、Top10、课表投影、终校验）
src/server/             → Fastify 后端
src/client/             → React 前端（Web 工作台）
tests/                  → 单测 / 契约测试 / 服务端测试 / E2E
```

- [ ] 没有调用 zdbk 写接口

- [ ] 硬约束逻辑在 `selection-model` 中，前端没有重复实现
